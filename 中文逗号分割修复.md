# 中文逗号分割修复

## 🐛 问题描述

**输入:**
```
28.08.38.31.04.各五，07.16.32.19.各十，14.三十五，澳门
```

**预期:**
应该识别出**3条记录**:
1. `28.08.38.31.04各五` → 5个号码,每个5元,总额25元
2. `07.16.32.19各十` → 4个号码,每个10元,总额40元
3. `14.三十五` → 1个号码,35元,总额35元

**实际(修复前):**
只识别了第1条"各五",后面的都被忽略了 ❌

---

## 🔍 原因分析

### 原来的逻辑:
```javascript
parseLine(line) {
    // 先尝试整体解析整行
    const parsed = this.tryParseAllRules(line);
    
    // 只有整体失败时才分割
    if (!parsed) {
        const segments = line.split(/[。，.]/);
        segments.forEach(...);
    }
}
```

**问题:**
- `28.08.38.31.04.各五，07.16.32.19.各十，14.三十五，澳门`
- 整体解析时,可能匹配到了第一条`28.08.38.31.04各五`
- 返回true,不再分割
- 后面的`07.16.32.19各十`和`14.三十五`被忽略

---

## ✅ 修复方案

### 新逻辑: 总是先分割,再逐条解析

```javascript
parseLine(line) {
    // 1. 先按分隔符分割成多个子句
    const segments = line.split(/[。，,.]/).filter(s => s.trim());
    
    // 2. 逐条解析每个子句
    segments.forEach(segment => {
        this.tryParseAllRules(segment.trim());
    });
}
```

**优势:**
- 无论整体是否能匹配,都会分割
- 每条子句独立解析
- 所有有效记录都会被识别

## 📊 支持的分隔符

| 分隔符 | 名称 | 示例 |
|--------|------|------|
| `。` | 中文句号 | `记录1。记录2` |

| `,` | 英文逗号 | `记录1,记录2` |

**注意:** 这些分隔符可以**混合使用**!

## 💡 核心改进

### 之前:
```
整体解析成功 → 返回,不分割
整体解析失败 → 分割后逐条解析
```

**问题:** 部分匹配会导致后面的记录被忽略

### 现在:
```
总是先分割 → 逐条解析 → 所有记录都能识别
```

**优势:** 不会遗漏任何有效记录
