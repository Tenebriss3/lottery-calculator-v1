# 百分号分隔符和单号格式修复

## 🐛 发现的问题

### 问题1: `%`无法作为号码分隔符
**输入:**
```
33%27%17%29%41%05%35%39%31各10米
```

**原因:**
- `extractNumbers`只识别`.`和空格作为分隔符
- `%`被忽略,导致号码提取失败

**修复:**
- 在`extractNumbers`中先将所有分隔符(`.` `%` `,` 空格)统一替换为空格
- 然后再提取数字

---

### 问题2: `25.=50米`解析错误
**输入:**
```
25.=50米
```

**你的意思:**
- 号码25,下单50元(应该是50米=50000元?)

**当前理解:**
- 正则匹配`(\d{1,2})[.=]\s*(\d+)(米|蚊|块|元)`
- 应该能正确识别为单号25,金额50000元

**如果还是有问题,可能是:**
- 需要先处理这行,避免被其他规则误匹配

---

## ✅ 修复内容

### 1. extractNumbers - 支持多种分隔符

```javascript
// 修复前
const matches = str.match(/\d{1,2}/g);

// 修复后
const normalized = str.replace(/[.%，,\s]+/g, ' ');
const matches = normalized.match(/\d{1,2}/g);
```

**现在支持的分隔符:**
- `.` (句点)
- `%` (百分号)
- `,` (逗号)
- `，` (中文逗号)
- ` ` (空格)

---

### 2. tryParseBatchBet - 正则支持`%`

```javascript
// 修复前
const batchPattern = /(?:[^0-9]*?)((?:\d{1,2}[.\s]+)+\d{1,2})\s*各\s*(\d+)(米|蚊|块|元)?/;

// 修复后
const batchPattern = /(?:[^0-9]*?)((?:\d{1,2}[.%\s]+)+\d{1,2})\s*各\s*(\d+)(米|蚊|块|元)?/;
```
