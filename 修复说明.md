# 解析问题修复说明

## 🐛 发现的问题

### 问题1: 特码格式无法识别
**输入:**
```
新奥： 特码01,02,06,07,08,09,10,13,15,16,20,21,22,28,45,49每个数20
```

**原因:** 
- 正则表达式要求必须有单位(米/蚊/块/元)
- "每个数20"没有单位,导致匹配失败

**修复:**
- 将单位改为可选 `(米|蚊|块|元)?`
- 没有单位时默认为"蚊"(元)

---

### 问题2: "各20"金额计算错误
**输入:**
```
22.34.46.12.24.48各20
```

**预期:**
- 6个号码,每个20元
- 总额 = 6 × 20 = 120元
- 每个号码统计 = 20元

**实际(修复前):**
- 总额被当作20元
- 每个号码 = 20 ÷ 6 = 3.33元 ❌

**原因:**
- 代码把"20"当作总额,然后除以号码数量
- 但"各20"的意思是"每个号码20元"

**修复:**
- 识别"各"关键字,理解为"每个号码的金额"
- 总额 = 每个号码金额 × 号码数量
- 统计时直接使用每个号码的金额

## 📊 金额计算规则总结

| 格式示例 | 含义 | 计算方式 |
|---------|------|---------|
| `平特蛇2000` | 生肖总额 | 总额=2000, 每个号码=2000÷4 |
| `包猪各25` | 生肖每个号码 | 每个=25, 总额=25×4 |
| `26.38.41各15米` | 批量每个号码 | 每个=15000, 总额=15000×3 |
| `22.34各20` | 批量每个号码(无单位) | 每个=20, 总额=20×2 |
| `特码01,02每个数10` | 特码每个号码 | 每个=10, 总额=10×2 |
| `25.=50米` | 单个号码 | 总额=50000, 只有一个号码 |

**关键理解:**
- **"各"** = 每个号码的金额
- **没有"各"** = 总额
- **没有单位** = 默认"蚊"(元)
- **"米"** = 1000元
