# 一行多条记录解析修复

## 🐛 问题描述

**输入:**
```
11.22.47.17.41.15.21.37.42.49.44.31.14.39各5米。包猪各25
```

**预期结果:**
应该识别出**2条记录**:
1. `11.22.47.17.41.15.21.37.42.49.44.31.14.39各5米` → 14个号码,每个5000元,总额70000元
2. `包猪各25` → 生肖猪,4个号码[08,20,32,44],每个25元,总额100元

**实际结果(修复前):**
只识别了第2条"包猪各25",第1条批量投注被忽略了 ❌

---

## 🔍 原因分析

### 原来的逻辑:
```javascript
parseLine(line) {
    if (tryParseZodiacBet(line)) return;  // 匹配到就返回
    if (tryParseBatchBet(line)) return;   // 前面的没匹配才试这个
    if (tryParseSingleBet(line)) return;
    if (tryParseSpecialCodeBet(line)) return;
}
```

**问题:**
- 一旦某个规则匹配成功,就立即return
- 同一行中的其他记录不会被解析
- "11.22...各5米。包猪各25" 整体匹配时,可能只匹配到"包猪各25"

---

## ✅ 修复方案

### 新逻辑: 分割后逐条解析

```javascript
parseLine(line) {
    // 1. 先尝试整体解析
    const parsed = this.tryParseAllRules(line);
    
    // 2. 如果整体解析失败,按句号/逗号分割后逐条解析
    if (!parsed) {
        const segments = line.split(/[。，.]/).filter(s => s.trim());
        
        segments.forEach(segment => {
            this.tryParseAllRules(segment.trim());
        });
    }
}
```

**优势:**
- 支持一行多条记录
- 用句号、逗号分隔的多条下注都能识别
- 每条记录独立解析,互不影响

## 📝 支持的行内分隔符

| 分隔符 | 示例 | 说明 |
|--------|------|------|
| `。` | `记录1。记录2` | 中文句号 |
| `，` | `记录1，记录2` | 中文逗号 |
| `.` | `记录1.记录2` | 英文句点 |
| `,` | `记录1,记录2` | 英文逗号 |

## 🎯 核心改进

### 之前:
```
一行只能识别一条记录
如果有多个,只识别第一个匹配成功的
```

### 现在:
```
一行可以识别多条记录
自动按分隔符拆分
每条独立解析
全部累加到结果中
```
